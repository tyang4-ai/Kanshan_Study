我是川大生物医学工程本科在读的一名学生，今春报了知乎 2026 黑客松，一个人。报名那天我桌上还摊着一本没翻几页的《信号与系统》，纸张已经有些泛黄，像是替我先衰老了一截。从 4 月下旬动工到今日提笔，14 个 phase，测试一千有余，倒也都过了——这便是「成绩」，是可以拿出来给人看的那种成绩。

可是我心里清楚，这桩活计与从前写课程作业，已不是同一回事了。

最大的不同，是我不再「写代码」。我做的事，是站在屏幕这一边，对着另一边的某种东西，指一指方向。

譬如前几日要给九只狐狸各配一份 prompt。若依旧时的写法，我大约要自己枯坐半夜，逐字推敲，再为某个莫名其妙的 bug 把 console.log 撒得到处都是。这一回我只说了一句「按 spec 各写一份，二百字以内，JSON-strict」，十分钟后那东西便把九份递了上来，我读毕，挑出两处不够具体的，又五分钟，事便了了。整个过程，我连那 prompt 模板的源码长什么样都未曾瞧上一眼。

这听上去像是「我什么也没做」。然而事情的另一面也立刻显现——某日它替我加了一段「自动重试 + 退避」的逻辑，单元测试照例是一片绿色，我险些便点了 commit。临了多看一眼 diff，方才看见它在 401 鉴权失败时也要重试，等于拿我的 API 额度去撞一堵关着的门。我让它删了。倘若那天偷懒少看一眼，这桩事大约要等到生产环境里钱包先空了一截，才会被人察觉。

所以「本科生用 AI 写代码省事」这句话，是骗人的。省下的是手指头的力气，添出来的是另一份警觉。我如今每日的时辰，大致是这样花的——三成在想清楚下一步要什么；三成在读它写出来的 diff，盯它是否在某个角落里偷偷埋雷；二成五在跑测试、看浏览器有无报错；剩下一成五，是与它来回拌嘴，敲定方向。

至于「写代码」这件事本身，已悄悄从我的日程上消失了。这倒也没什么。

只是有一桩事，我至今想不太明白：这样的活法，毕业以后还有没有去处？我能想到的也无非两条路——一条是，到了 2027 年，人人都这么干，谁还坚持手敲，便算是错过了一辆车；另一条是，这不过是一段甜蜜的窗口期，等所有应届生都会用 Claude Code，「会用 AI」便不再是本事，「想得清楚要什么」才是本事，而后者与会不会用 AI，原是两码事。

我自己押的是后者。这也是我做看山书房时未曾明说的私心——倘若「会写代码」这件事终要贬值，那么「拼得出一套稳定的多 agent 工作流」便还值几个钱。不过这话似乎也只能我自己说，且说给自己听。

这个周末，我那门概率论的作业还须交，我打算继续手写。AI 替我做选择题尚可，可解题的思路不像 React 组件那般有现成的模板，它给出的步骤常是对一半错一半，自己捋一遍反倒来得快。
